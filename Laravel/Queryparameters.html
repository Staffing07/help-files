Query parameters

Can be used to detect where a user has visited the site from.

pizzahouse.com/?source=facebook&campaign=half_price_promo

In above example, we have an advert on facebook for our store that directs users to the website via this query parameter.

? denotes beginning of query string

& denotes another parameter in string

Essentitally the user will end up at the same site, however the server can detect the query parameters in the url to tailor a specific experience based upon what the query parameter was

For example, our site would now give users half price pizzas because they visited via this url.


Example

Route::get('/recipes', function () { // get data from database // pass data as an array into the view $pizzas = [ 'type' => 'hawaiian', 'base' => 'deep pan', 'cuisine' => 'Italian' ]; $name = request('name'); return view('recipes', [ 'pizzas' => $pizzas, 'name' => $name ]); });

$name = request('name');

The url request is attached using this line.

Where the url includes a "name" parameter within the recipes route /recipes?name=fred

The value of name will be stored in the $name variable.

This $name variable can then be referenced in the view.

return view('recipes', [ 'pizzas' => $pizzas, 'name' => $name ]);

Then in the view file you just need to include

{{ $name }}

Example 2

Routes file

return view('recipes', [ 'pizzas' => $pizzas, 'name' => request('name'), 'age' => request('age') ]);

View file

{{ $name }} {{ $age }}
