What is Laravel?

  • Framework for creating php websites
  • Works well with MySQL databases
  • Includes useful features such as (blade templating engine, auth (authentication module), eloquent (used to query SQL databases))

Uses Model View Controller pattern

  • Model = database
  • View = rendered webpage
  • Controller = The thing in the middle that takes the data from the database and converts it to the webpage

Setup

Need to install:

  • xampp - for test server on localhost
  • composer - a package or dependency manager for php that installs php packages into projects (similar to npm in node)

Close terminal and vscode and reopen

Type composer -V into terminal. If don't get version number not installed correctly.

Type composer global require laravel/installer

Navigate to folder where you want to create the Laravel app and type laravel new nameofproject

Once installed cd into newly created project folder and type php artisan serve

This spins up a dev server to preview the project.

How does a Laravel site work?

Request from url comes in

If a route file matches the Url requested, this route file will tell the browser what to do next.

This could be something like, modify a database in some way, retrieve data or display a page.

Then the view is compiled and returned to the browser.

  • Users page requested in url
  • Route file matching users runs code to get users list from database and then render it using a view to a page for the user.

Laravel files

routes/web.php

This file shows all potential urls with their return views that are in the resource/views folder.

These views are templates, so think of them as the page layout with placeholders for other things in them.
