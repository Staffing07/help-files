How does a Laravel site work?

Request from url comes in

If a route file matches the Url requested, this route file will tell the browser what to do next.

This could be something like, modify a database in some way, retrieve data or display a page.

Then the view is compiled and returned to the browser.

  • Users page requested in url
  • Route file matching users runs code to get users list from database and then render it using a view to a page for the user.

Laravel files

routes/web.php

This file shows all potential urls with their return views that are in the resource/views folder.

These views are templates, so think of them as the page layout with placeholders for other things in them.

Route::get('/', function () { return view('welcome'); });

This route will trigger the homepage view.

The 'welcome' accesses the resources/views/welcome blade layout

Route::get('/recipes', function () { return view('welcome'); });

This route will trigger the recipes view.

Route return types

You can return different types

Route::get('/recipes', function () { return 'recipes'; });

This would return a page showing plain text/html with the word recipes rather than using a view file.

Route::get('/recipes', function () { return ['name' => 'tomato soup', 'ingredient' => 'tomatoes']; });

This would return a json string object.

Using blade to pass data from the route file to the view

Route::get('/recipes', function () { return view('recipes', ['cuisine' => 'Indian' ]); });

The above example will return the recipes view when the user visits /recipes. We are also passing an array through to the view using the variable 'cuisine'

In the view file we could then type {{ $cuisine }} to pull that data value into the view file

The double curly braces also escapes any special characters.

A more complex array

$pizzas = [ ['type' => 'hawaiian', 'base' => 'cheesy crust'], ['type' => 'volcano', 'base' => 'garlic crust'], ['type' => 'veg supreme', 'base' => 'thin & crispy crust']
